1. 객체지향

   1) 메서드(Method)
    - 어떤 작업을 수행하는 코드들의 집합
      = 작업의 최소 단위
    - 선언부(Header)와 구현부(Body) 로 구분하여 정의
    - 메서드는 반드시 호출되어야만 실행될 수 있다.

      => 이 때, 호출하는 메서드를 Caller 메서드라고 하며
         호출 당해서 작업을 수행하는 메서드를 Worker 메서드라고 한다. 
    - 메서드 정의 시 리턴값이 없을 경우, 리턴타입을 생략할 수 없으며
      반드시 void 라는 특수한 타입을 명시해야한다! (리턴값이 없다는 의미)

    - < 메서드 정의 기본 문법 >
     [제한자] 리턴타입 메서드명([매개변수...]) { // 선언부(Header)
    // 메서드 중괄호(구현부 = Body) 내에 호출 시 수행할 작업들을 기술...
     }
     
    2) 형태에 따른 분류
    - 매개변수도 없고, 리턴값도 없는 메서드
    => 매개변수가 없으므로 소괄호() 안에 아무것도 명시하지 않음
	=> 리턴값이 없으므로 리턴타입에 void 를 명시

    - 매개변수는 없고, 리턴값만 있는 메서드
    => 매개변수가 없으므로 소괄호() 안에 아무것도 명시하지 않음
    => 리턴값이 있으므로 리턴타입에 리턴할 데이터("물")의 타입(String)을 명시
    
    - 매개변수만 있고, 리턴값은 없는 메서드
    => 매개변수가 있으므로, 전달받을 데이터를 저장할 변수를 소괄호() 안에 선언
    => 리턴값이 없으므로 리턴타입에 void 를 명시

    - 매개변수도 있고, 리턴값도 있는 메서드

1.매서드

            
1) 어떤 작업을 수행하는 코드들의 집합 = 작업의 최소단위

2) 선언부와 구현부로 구분하여 정의

3) 반드시 호출해야만 실행될 수 있다. 

4) 간단한 개략도

 - 호출하는 매서드를 caller, 작업수행 매서드를 worker라 함
 - 리턴값이 없을 경우 return타입을 생략할 수 없다. 
 - 반드시 void 라는 특수타입 설정(리턴값이 없다는 의미)

5) 리턴타입은 한가지만 명시, 리턴할 수 있는 값도 한가지만

6) 정의 기본 문법
 [제한자] 리턴타입 매서드명([매개변수...]) {
     //선언부
     구현부 내에 호출할 작업을 기술 
     [return [값];]
 }

 7) 형태에 따른 분류
 - 매개변수도 없고, 리턴값도 없는 메서드
 - 매개변수는 없고, 리턴값만 있는 메서드
 - 매개변수만 있고, 리턴값은 없는 메서드
 - 매개변수도 있고, 리턴값도 있는 메서드
 ----------------------------------------
 - 매개변수가 2개 이상인 메서드(3번 또는 4번 형태 모두 사용 가능)
